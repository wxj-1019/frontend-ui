# Frontend UI 网站设计方案

> 大胆前卫 · 不拘一格 · 动画驱动的沉浸式体验

---

## 一、设计定位

### 1.1 设计理念

**"代码即艺术"** — 将组件库网站本身打造成一个动画艺术作品，让用户在浏览文档时就能感受到组件的强大表现力。

### 1.2 设计原则（来自 Design Skills）

> 基于 ui-design-brain + frontend-design skills 提炼的核心原则：

```
1. 克制优于装饰（Restraint over decoration）
   ├── 更少的元素，更高的精致度
   ├── 留白是特性，不是空白
   └── 首页大胆，文档区克制

2. 排版承载个性（Typography carries personality）
   ├── Space Grotesk（几何感标题）+ Outfit（可读正文）
   ├── 最大化标题与正文的字重对比
   └── 排版本身是设计记忆点，不是中性载体

3. 一个强色彩瞬间（One strong color moment）
   ├── 中性色盘优先（深空黑 + 暗灰层次）
   ├── 霓虹色仅作点缀强调
   └── 如果它能出现在海报上，可能是永恒的

4. 间距即结构（Spacing is structure）
   ├── 4px 基础网格
   ├── 紧凑间距 → 分组关联元素
   └── 宽松间距 → 让 Hero 内容呼吸

5. 可访问性不可协商（Accessibility is non-negotiable）
   ├── WCAG AA 对比度最低标准
   ├── 所有交互元素有 focus 指示器
   ├── 语义化 HTML + ARIA
   └── 键盘导航完整支持

6. 拒绝通用 AI 美学（No generic AI aesthetics）
   ├── ❌ 紫白渐变
   ├── ❌ 均匀间距卡片网格
   ├── ❌ cookie-cutter 布局
   └── ✅ 每个界面为特定场景而设计
```

### 1.2 差异化风格

| 维度 | React Bits | Aceternity | **Frontend UI** |
|------|-----------|------------|-----------------|
| 整体风格 | 简约暗色 | 现代渐变 | **克制的赛博朋克** |
| 色彩 | 单一暗色 | 渐变彩色 | **高对比霓虹（作强调色）** |
| 布局 | 传统网格 | 网格+卡片 | **首页不对称 / 文档区标准** |
| 交互 | 基础悬停 | 微交互 | **局部光标追踪+视差** |
| 转场 | 无 | 基础 | **首页级动画转场** |
| 氛围 | 工具感 | 设计感 | **首页艺术装置 / 文档区精密** |

### 1.3 定位分层（核心约束）

> ⚠️ 组件库文档站的首要目标是「快速找到组件 → 看预览 → 复制代码」，
> 沉浸式体验是加分项而非阻碍项。因此本方案对两个区域采用**完全不同的设计密度**：

| 区域 | 设计目标 | 布局策略 | 动效密度 |
|------|---------|---------|--------|
| **首页 / Landing** | 震撼第一印象，展示组件库实力 | 不对称、沉浸式、艺术装置感 | 满配：开场串行动画、粒子、3D宇宙 |
| **文档区** | 零摩擦检索、高频重复访问 | 固定导航 + 侧边栏 + 内容三栏（标准布局） | 克制：仅微交互 + 进场动画，无全局转场 |
| **分类页** | 快速浏览对比 | 网格化卡片，每卡带迷你预览 | 中等：hover 放大发光 |

**核心原则**：导航栏、侧边栏位置在文档区**永远不变**，用户不会迷路。
霓虹色仅作强调色（链接、激活态、CTA），正文背景保持深空中性色，避免视觉疲劳。

### 1.4 质量标准（Quality Bar）

> 来自 ui-design-brain skill：输出应匹配顶级 SaaS 公司高级产品设计师的水准：

```
质量标准：
├── ✅ 清晰的视觉节奏，有意识的不对称
├── ✅ 明显的交互暗示（hover、focus、active 状态）
├── ✅ 优雅的边界情况（空状态、加载态、错误态）
├── ✅ 响应式无断点伪影
├── ✅ 文案：动词优先（"保存更改" 而非 "提交"）
├── ✅ 每个页面只有一个主操作按钮
└── ✅ 禁止：彩虹徽章、模态框嵌套、无解释的禁用按钮
```

### 1.5 反模式清单（Anti-Patterns）

> 来自 ui-design-brain skill：禁止出现以下设计：

```
反模式（禁止）：
├── ❌ 彩虹徽章 — 每个状态不同亮色，无语义意义
├── ❌ 模态框内嵌模态框 — 用页面或抽屉替代
├── ❌ 禁用按钮无解释 — 必须说明缺少什么
├── ❌ 已知布局用 Spinner — 改用骨架屏
├── ❌ "点击这里" 链接 — 链接文本必须描述目标
├── ❌ 桌面端汉堡菜单 — 有空间时显示可见导航
├── ❌ 自动轮播 — 让用户控制导航
├── ❌ 仅占位符表单 — 始终使用可见标签
├── ❌ 等权重按钮 — 建立主/次/三级层次
├── ❌ 超小文字 (< 12px) — 正文最小 14px，推荐 16px
└── ❌ 模板化 AI 美学 — 拒绝紫白渐变、均匀网格
```

## 二、视觉设计系统

### 2.1 色彩系统

> 来自 frontend-design skill："一个强色彩瞬间" — 中性色盘优先，霓虹色仅作点缀。

```
主色调（霓虹系列 — 仅作强调色）
├── 主色：#00F5FF (Cyan) — 主要强调色、链接、激活态
├── 辅助：#FF006E (Magenta) — 次要强调色、警告、删除
├── 第三：#8B5CF6 (Violet) — 渐变过渡色、次要信息
└── 成功：#00FF88 (Neon Green) — 状态色、成功提示

背景色（深空系列 — 中性色盘）
├── 主背景：#0A0A0F — 近纯黑带蓝调（页面底色）
├── 次背景：#12121A — 卡片/区块背景
├── 表面：#1A1A2E — 悬浮层背景、侧边栏
└── 边框：#2A2A3E — 分割线/边框

文字色（均满足 WCAG AA 4.5:1 对比度，背景 #0A0A0F）
├── 主文字：#FFFFFF — 标题/重点（对比度 19.3:1 ✅）
├── 次文字：#A8A8BC — 正文/描述（对比度 7.2:1 ✅）
└── 弱文字：#8A8A9E — 辅助信息（对比度 5.1:1 ✅）

> ⚠️ 禁止使用 #606070 以下灰度作为可读文字（对比度仅 3.3:1，不达标）

### 2.1.1 霓虹强调色使用约束
霓虹色（青/品红/绿）**仅用于**：链接、激活态、CTA 按钮、图标、装饰渐变。
霓虹色**不得用于**：大面积背景、正文、表格条纹、表单标签。
（理由：高饱和霓虹大面积铺陈会引发视觉疲劳，且品红 #FF006E 与绿 #00FF88 组合对红绿色盲不友好）

### 2.1.2 交互状态色规范

| 状态 | 色彩 | 用途 |
|------|------|------|
| **默认** | #2A2A3E | 边框、分割线 |
| **悬停** | #00F5FF (Cyan) | 链接悬停、按钮边框 |
| **激活/聚焦** | #00F5FF + glow | 按钮点击、输入框聚焦 |
| **选中** | #8B5CF6 (Violet) | 侧边栏当前项、标签页 |
| **成功** | #00FF88 | 表单验证通过、操作成功 |
| **警告** | #FFB800 | 注意事项、非阻断提示 |
| **错误/危险** | #FF006E (Magenta) | 错误状态、删除操作 |
| **禁用** | opacity 0.4 | 不可交互元素 |
```

### 2.2 渐变系统

```css
/* 主渐变 */
--gradient-primary: linear-gradient(135deg, #00F5FF 0%, #8B5CF6 50%, #FF006E 100%);

/* 文字渐变 */
--gradient-text: linear-gradient(90deg, #00F5FF, #FF006E);

/* 边框渐变 */
--gradient-border: linear-gradient(135deg, #00F5FF, #8B5CF6, #FF006E);

/* 发光效果 */
--glow-cyan: 0 0 20px rgba(0, 245, 255, 0.5);
--glow-magenta: 0 0 20px rgba(255, 0, 110, 0.5);
```

### 2.3 字体系统（已定稿）

> 来自 frontend-design skill："排版承载个性" — 最大化标题与正文的字重对比。

```
字体配对：
├── 标题字体：Space Grotesk — 几何感强、个性鲜明
│   ├── 用途：Hero/H1/H2/品牌标题
│   ├── 字重：700-800（粗体强调）
│   └── 特点：等宽字母、科技感
├── 正文字体：Outfit — 屏幕可读性佳
│   ├── 用途：正文/描述/UI 文字/标签
│   ├── 字重：400（正文）/ 500（强调）/ 600（小标题）
│   └── 特点：开源、可读性高、字重丰富
└── 代码字体：JetBrains Mono — 编程连字支持
    ├── 用途：代码块/行内代码/命令
    ├── 字重：400
    └── 特点：连字、区分 0/O、区分 l/1

字号层级（基于 1.25 模数比）
├── Hero：6xl-8xl (60-96px) — 超大标题，letter-spacing: -0.02em
├── H1：4xl-5xl (36-48px)
├── H2：2xl-3xl (24-36px)
├── H3：xl (20px)
├── 正文：base (16px) — line-height: 1.5
├── 小字：sm (14px)
└── 微字：xs (12px) — 仅用于辅助标签

排版规范：
├── 最大行宽：65-75 字符（正文可读性）
├── 行高：1.5（正文）/ 1.2-1.3（标题）
├── 字间距：-0.01em（大标题）/ 0（正文）
├── 每页只有一个 h1
└── 标题层级严格：h1 → h2 → h3
```

### 2.5 设计 Token 规范

> 来自 ui-design-brain skill："颜色用 CSS 变量保持一致性"。
> 所有颜色、字体、间距必须通过**语义化 Token** 引用，**禁止在业务代码中直接写 hex 值**。
> 这保证暗/亮主题可一键切换，且全站视觉一致。

#### 2.5.1 Token 命名约定（语义化）

```css
/* 背景层 */
--color-bg-primary:     #0A0A0F;  /* 主背景 */
--color-bg-secondary:   #12121A;  /* 卡片/区块 */
--color-bg-surface:     #1A1A2E;  /* 悬浮层 */
--color-bg-elevated:    #222236;  /* 弹窗/下拉 */

/* 文字层 */
--color-text-primary:   #FFFFFF;  /* 主文字 */
--color-text-muted:     #A8A8BC;  /* 次文字 */
--color-text-subtle:    #8A8A9E;  /* 弱文字 */
--color-text-disabled:  #5A5A6E;  /* 禁用文字 */

/* 强调色（霓虹，仅作点缀） */
--color-accent:    #00F5FF;
--color-accent: #FF006E;
--color-accent:  #8B5CF6;
--color-success: #00FF88;
--color-warning: #FFB800;

/* 边框/分割 */
--color-border-default: #2A2A3E;
--color-border-subtle:  rgba(255,255,255,0.06);
--color-border-strong:  rgba(255,255,255,0.12);

/* 交互状态 */
--color-hover:          rgba(0, 245, 255, 0.1);  /* 悬停背景 */
--color-focus-ring:     rgba(0, 245, 255, 0.5);  /* 焦点环 */
--color-active:         rgba(0, 245, 255, 0.15); /* 激活背景 */

/* 字体 */
--font-display:  'Space Grotesk', sans-serif;  /* 标题 */
--font-sans:     'Outfit', sans-serif;          /* 正文 */
--font-mono:     'JetBrains Mono', monospace;  /* 代码 */

/* 阴影/发光 */
--shadow-sm:     0 1px 2px rgba(0,0,0,0.3);
--shadow-md:     0 4px 12px rgba(0,0,0,0.4);
--shadow-lg:     0 8px 24px rgba(0,0,0,0.5);
--glow-cyan:     0 0 20px rgba(0, 245, 255, 0.3);
--glow-magenta:  0 0 20px rgba(255, 0, 110, 0.3);

/* 圆角 */
--radius-sm:     4px;
--radius-md:     8px;
--radius-lg:     12px;
--radius-xl:     16px;
--radius-full:   9999px;

/* 动画时长 */
--duration-fast:     100ms;
--duration-normal:   200ms;
--duration-slow:     300ms;
--duration-slower:   500ms;

/* Easing */
--ease-out:      cubic-bezier(0.16, 1, 0.3, 1);
--ease-in:       cubic-bezier(0.4, 0, 1, 1);
--ease-in-out:   cubic-bezier(0.4, 0, 0.2, 1);
--ease-spring:   cubic-bezier(0.34, 1.56, 0.64, 1);
```

#### 2.5.2 暗/亮主题映射

```css
/* 暗色主题（默认） */
:root {
  --color-bg-primary:   #0A0A0F;
  --color-bg-secondary: #12121A;
  --color-bg-surface:   #1A1A2E;
  --color-text-primary: #FFFFFF;
  --color-text-muted:   #A8A8BC;
  --color-text-subtle:  #8A8A9E;
  --color-border-default: #2A2A3E;
}

/* 亮色主题 */
:root[data-theme="light"] {
  --color-bg-primary:   #FAFAFA;
  --color-bg-secondary: #FFFFFF;
  --color-bg-surface:   #F0F0F5;
  --color-bg-elevated:  #FFFFFF;
  --color-text-primary: #0A0A0F;
  --color-text-muted:   #4A4A5E;
  --color-text-subtle:  #6A6A7E;
  --color-text-disabled: #9A9AAE;
  --color-border-default: #E5E5EA;
  --color-border-subtle: rgba(0,0,0,0.06);
  --color-hover:        rgba(0, 150, 200, 0.08);
  --color-focus-ring:   rgba(0, 150, 200, 0.5);
  --color-active:       rgba(0, 150, 200, 0.12);
}

/* 系统偏好自动切换 */
@media (prefers-color-scheme: light) {
  :root:not([data-theme="dark"]) {
    /* 继承亮色主题变量 */
  }
}
```

#### 2.5.3 使用规则
- ✅ 业务代码用 `bg-[var(--color-bg-primary)]` 或 Tailwind 别名 `bg-bg-primary`
- ❌ 禁止 `bg-[#0A0A0F]`、`text-white`（裸值不随主题切换）
- ✅ 添加新颜色时先补 Token，再使用
- ✅ 交互状态用语义 Token（--color-hover、--color-focus-ring）
- ❌ 禁止直接写 `rgba()` 硬编码值

### 5.1 全局光标效果

> ⚠️ 实现必须使用 `gsap.quickTo()` 而非普通 tween，避免高频更新导致性能问题。

```
效果：自定义光标 + 轨迹粒子
├── 默认状态：小圆点 (8px) + 外圈跟随 (32px)
├── 悬停链接：放大 + 霓虹发光
├── 点击：涟漪扩散效果
└── 移动：拖尾粒子（3-5个，逐渐缩小消失）

技术实现：
├── 光标跟随：gsap.quickTo()（60fps 性能关键）
├── 涟漪效果：CSS @keyframes（transform + opacity）
├── 减弱动效：完全关闭自定义光标，保留系统光标
└── 触摸设备：不渲染自定义光标，仅触发触摸涟漪
```

**光标跟随代码示例**：
```tsx
// 使用 quickTo 实现 60fps 光标跟随
useGSAP(() => {
  const xTo = gsap.quickTo(cursorRef.current, "x", {
    duration: 0.4,
    ease: "power3",
  });
  const yTo = gsap.quickTo(cursorRef.current, "y", {
    duration: 0.4,
    ease: "power3",
  });

  const onMove = (e: MouseEvent) => {
    xTo(e.pageX);
    yTo(e.pageY);
  };

  window.addEventListener("mousemove", onMove);
  return () => window.removeEventListener("mousemove", onMove);
});
```

### 5.4 微交互动画

> 参考 Disney 12 原则 + UI 动画最佳实践，所有微交互必须遵循以下规范：

```
微交互时间规范：
├── 悬停 (hover)：150-250ms ease-out
├── 点击 (active)：50-100ms（即时反馈）
├── 聚焦 (focus)：200ms ease-out
├── 页面转场：300-500ms
├── 复杂揭示：500-800ms
└── 禁止超过 1000ms（用户会感到延迟）
```

**具体微交互规范**：

| 元素 | 状态 | 效果 | 数值 |
|------|------|------|------|
| **按钮** | hover | 边框霓虹流动 + 微弹 | scale(1.02), 200ms ease-out |
| **按钮** | active | 缩小→弹回 + 涟漪 | scale(0.97), 80ms ease-in |
| **卡片** | hover | 3D 倾斜 + 光泽移动 | scale(1.01), translateY(-2px), 250ms |
| **输入框** | focus | 边框渐变 + 标签上浮 | 200ms cubic-bezier(0.4, 0, 0.2, 1) |
| **开关** | toggle | 弹性滑动 + 颜色渐变 | 250ms cubic-bezier(0.34, 1.56, 0.64, 1) |
| **标签页** | switch | 下划线滑动 + 内容交叉淡入 | 300ms ease-out |

**Easing 速查**：
```css
/* 进入动画 */
transition: transform 300ms cubic-bezier(0.16, 1, 0.3, 1); /* 弹性进入 */

/* 退出动画 */
transition: opacity 200ms cubic-bezier(0.4, 0, 1, 1); /* 平滑退出 */

/* 弹簧回弹 */
transition: transform 250ms cubic-bezier(0.34, 1.56, 0.64, 1); /* 过冲回弹 */

/* 平滑通用 */
transition: all 300ms cubic-bezier(0.4, 0, 0.2, 1); /* Material 标准 */
```

**Anticipation（预备动作）**：
- 按钮点击前文字微移 1-2px
- Modal 打开前先回退 10px
- 菜单项展开前先收缩再展开

### 5.5 减弱动效降级（可访问性，必须实现）

> ⚠️ 本站动画密度高，**必须**尊重用户的 `prefers-reduced-motion` 系统设置，
> 否则会让晕动症/前庭功能障碍用户产生不适。这是上线前的硬性要求。

#### 检测用户偏好
```css
/* 全局降级：用户勾选「减弱动效」时生效 */
@media (prefers-reduced-motion: reduce) {
  *, *::before, *::after {
    animation-duration: 0.01ms !important;
    animation-iteration-count: 1 !important;
    transition-duration: 0.01ms !important;
    scroll-behavior: auto !important;
  }
}
```

#### 各动画模块的降级策略
| 动画模块 | 默认表现 | 减弱动效下 |
|---------|---------|-----------|
| 背景粒子（Particles） | 持续运行 | **完全禁用**（不渲染 Canvas）|
| 页面几何转场 | 圆形扩散/收缩 | 简化为即时切换（0ms）|
| 多层视差滚动 | 0.3x/0.6x/1x | 统一 1x（无视差）|
| 打字机开场 | 逐字打出 | 直接显示完整文本 |
| 自定义光标轨迹 | 拖尾粒子 | 保留系统光标 |
| 卡片 3D 倾斜 hover | 倾斜跟随 | 仅边框高亮 |
| 交错进场 stagger | 依次入场 | 同时入场（仅淡入）|

#### Motion 引擎实现示例
```tsx
import { useReducedMotion } from 'motion/react';

function FadeContent({ children }) {
  const shouldReduce = useReducedMotion();
  return (
    <motion.div
      initial={{ opacity: 0, y: shouldReduce ? 0 : 20 }}
      animate={{ opacity: 1, y: 0 }}
      transition={{ duration: shouldReduce ? 0 : 0.5 }}
    >
      {children}
    </motion.div>
  );
}
```

## 八、性能预算与优化

### 8.1 性能预算（量化指标，上线红线）

> 本站动画密集，必须设定明确的性能红线，任何 PR 不得退步以下指标：

| 指标 | 红线 | 测量工具 | 说明 |
|------|------|---------|------|
| LCP（最大内容绘制） | < 2.5s | Lighthouse | 首页 Hero 文字渲染完成 |
| TBT（总阻塞时间） | < 200ms | Lighthouse | 首屏 JS 执行不阻塞主线程 |
| CLS（布局偏移） | < 0.1 | Lighthouse | 动画不得引起布局抖动 |
| 首屏 JS Bundle | < 200KB gzip | @next/bundle-analyzer | 不含 GSAP/Three.js |
| 动画帧率 | ≥ 55fps | DevTools Performance | 滚动/hover 时 |
| INP（交互延迟） | < 200ms | 真实用户监控 | 点击/输入响应 |

### 8.2 视口外暂停策略（关键）

> ⚠️ 当前 `Particles` 组件是 `fixed inset-0` 全屏常驻渲染，
> 即使用户滚到页脚仍在烧 GPU。**必须按以下策略改造所有持续型动画：**

```tsx
// Canvas 动画必须用 IntersectionObserver 守卫
function Particles() {
  const [inView, setInView] = useState(true);
  // 当容器离开视口，停止 requestAnimationFrame
  // 标签页隐藏（document.hidden）时也暂停
}
```

| 动画类型 | 默认运行条件 | 暂停条件 |
|---------|-------------|----------|
| 背景粒子 Canvas | 容器在视口内 | 离开视口 / 标签页隐藏 |
| 3D 组件宇宙 | 首页可视区域 | 滚出首页 / reduced-motion |
| 全局光标轨迹 | 鼠标在页面上 | 标签页隐藏 |
| Lenis 平滑滚动 | 始终 | reduced-motion 时关闭 |

### 8.3 动画渲染性能

```
核心原则：只动画 transform / opacity（合成层属性）

✅ 允许动画的属性：
├── transform: translateX/Y/Z, scale, rotate, rotateX/Y
├── opacity
├── filter（已合成时）
└── clip-path

❌ 禁止动画的属性（触发重排/重绘）：
├── width / height
├── top / left / right / bottom
├── margin / padding
├── border-width
└── font-size

性能规范：
├── 合成层数量限制 < 30 个/页面（过多会增加内存）
├── will-change 仅加在真正会动画的元素上，动画后移除
├── 动画容器加 contain: layout style paint
├── backdrop-filter 仅限固定定位元素（如导航栏）
├── box-shadow 大半径会降低性能，改用 border 或伪元素
└── filter: blur() 触发重绘，优先使用预模糊图片
```

### 8.4 加载性能

```
首屏加载策略：
├── 首屏只加载关键 CSS/JS（首屏 JS < 200KB gzip）
├── 背景动画延迟加载（idle callback 后初始化）
├── 组件预览使用懒加载（next/dynamic）
├── GSAP / Three.js 必须按需 dynamic import，不进首屏 bundle
├── 图片使用 Next.js Image 组件（自动 webp + 响应式）
├── 字体使用 next/font 优化（避免 FOUT）
├── 开场动画仅首次访问触发（sessionStorage 标记）
├── 关键路径 CSS 内联
└── Loading 状态：300ms 后才显示（避免快速加载闪烁）

骨架屏规范：
├── 使用 Skeleton 替代 Spinner（已知布局时）
├── 骨架屏动画：subtle shimmer（200ms 循环）
├── 骨架屏颜色：bg-secondary 色调
└── 加载完成后 crossfade 过渡到实际内容
```

### 8.5 首次访问缓存策略

```
沉浸式开场动画流程控制：
├── 首次访问：播放完整 6 段开场动画（粒子汇聚 + 打字机...）
├── 重复访问：跳过开场，直接展示内容（sessionStorage 标记）
├── 文档区跳转：绝不重播开场（只播 1 次）
└── 减弱动效：所有开场简化为即时显示
```

> （红绿色盲无法区分），必须额外加图标/文字辅助。

### 9.2 键盘导航

```
键盘导航规范：
├── 所有交互元素可通过 Tab 到达，顺序与视觉顺序一致
├── 每个可聚焦元素必须有可见的 focus 指示器（2-4px 发光环 + offset）
├── 使用 :focus-visible 而非 :focus（避免鼠标点击也显示焦点环）
├── Escape 可关闭模态框/搜索框（⌘K）
├── 提供「跳到主内容」的跳转链接（首次 Tab 出现）
└── Modal 内容必须实现 focus trap（焦点锁定在弹窗内）

Focus 环样式：
├── 宽度：2-4px
├── 颜色：brand accent（Cyan）或白色 80% opacity
├── 偏移：2-4px（不与元素边框重叠）
└── 减弱动效：仍需保留可见 focus indicator
```

### 9.3 触摸目标尺寸（WCAG 2.5.8）

```
所有交互元素最小尺寸：
├── 触摸目标：24x24px（WCAG 最小要求）
├── 推荐触摸目标：44x44px（iOS HIG）
├── 按钮最小高度：40px
├── 链接最小点击区域：24px（含 padding 扩展）
└── 图标按钮：需额外 padding 扩大点击区域
```

### 9.4 自定义光标兼容性
> 自定义光标（5.1 节）会隐藏系统光标，破坏键盘/辅助技术用户体验。策略：
- **不隐藏**原生光标（不设 `cursor: none`），自定义光标作为**增强层**叠加
- 用户启用屏幕阅读器时，自动隐藏自定义光标层
- 减弱动效模式下，完全关闭自定义光标
- 触摸设备不渲染自定义光标，仅触发触摸涟漪

### 9.5 动态内容 ARIA

```
动态内容必须通知屏幕阅读器：
├── 搜索结果更新：aria-live="polite"
├── 组件切换：aria-live="assertive"
├── 加载状态：aria-busy="true"
├── 错误信息：aria-describedby 关联输入框
├── 展开/折叠：aria-expanded
└── 模态框：role="dialog" + aria-modal="true"
```

### 9.6 语义化与 ARIA
- 导航用 `<nav>`、侧边栏用 `<aside>`、主内容用 `<main>`
- 图标按钮必须有 `aria-label`
- 组件预览区用 `role="region"` + `aria-label`
- 表单控件关联 `<label>`

### 10.2 动画引擎策略（核心）

> ⚠️ 多引擎共存不能全部打进首屏。采用「默认引擎 + 按需加载」策略：

| 引擎 | 定位 | 何时加载 | 示例 |
|------|------|---------|------|
| **Motion（默认）** | 90% 场景：进场/hover/手势 | 首屏 bundle | BlurText、FadeContent、Dock |
| **GSAP** | 高级场景：ScrollTrigger、SplitText、时间线 | 路由级 dynamic import | 首页开场动画、组件宇宙 |
| **Three.js** | 3D 场景 | 仅在组件宇宙页加载 | 6.1 节组件宇宙 |
| **Lenis** | 平滑滚动 | 首屏（可延后） | 全局 |

```tsx
// GSAP / Three.js 必须按需加载，不进首屏 bundle
const ComponentUniverse = dynamic(() => import('./ComponentUniverse'), {
  ssr: false,
  loading: () => <Skeleton />,
});
```

**禁止**：在 `packages/ui` 的默认导出（`src/index.ts`）中直接 import gsap/three，
否则会污染所有使用该库的页面的首屏。重型引擎组件走独立子路径导出：
`@frontend-ui/ui/advanced/*`。
