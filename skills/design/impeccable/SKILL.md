# Impeccable

Flagship design skill for crafting production-grade interfaces that avoid generic AI aesthetics. Every decision is intentional and defensible.

## When to Use

- Building a hero/landing page
- Designing a key product feature UI
- User explicitly asks for "polished" or "premium" design
- Any interface that will be customer-facing

## Anti-Generic Rules

### Never Do These
- ❌ Purple/blue gradient hero backgrounds
- ❌ Inter or Roboto as default font (unless explicitly chosen)
- ❌ Even 3-column card grids with equal heights
- ❌ "Revolutionize your workflow" headlines
- ❌ Centered text for anything longer than one line
- ❌ Box shadows on every card
- ❌ Rounded corners everywhere (pick a strategy)

### Always Do These
- ✅ Intentional asymmetry in layout
- ✅ One strong accent color, used sparingly
- ✅ Distinctive font pairing (display + body)
- ✅ Real copy, not placeholder lorem ipsum
- ✅ White space as an active design element
- ✅ Visual hierarchy clear at a glance

## Design Process

### 1. Define the Subject
- What is this page about? Name it concretely.
- Who is the audience? What do they care about?
- What is the ONE action the page needs to drive?

### 2. Build a Token System
```
Palette: 5-6 named colors (1 background, 1 foreground, 1 accent, 2-3 supporting)
Type: 1 display face (used sparingly) + 1 body face + optional 1 utility face
Spacing: Define scale (4/8/12/16/24/32/48/64/96)
Radius: One consistent strategy (none / small 4px / medium 8px / large 16px)
```

### 3. Before Writing Code
- Critique your own plan against the brief
- Ask: "Would a human designer make this specific choice for this specific brief?"
- If any part feels generic, revise it and explain why you changed it

## Escalation

If uncertain about a design direction, present 2-3 brief options to the user:
- Option A: Direction + 3-word rationale
- Option B: Different direction + 3-word rationale
- Ask which they prefer before building

## Quality Checklist

Post-build, verify:
- [ ] No purple/gradient defaults
- [ ] Font pairing is intentional, not Inter/Roboto
- [ ] Layout has intentional asymmetry
- [ ] Accent color used in ≤3 places
- [ ] White space creates clear visual groups
- [ ] Mobile layout is considered, not an afterthought
- [ ] Focus states are visible
- [ ] Reduced motion is respected
