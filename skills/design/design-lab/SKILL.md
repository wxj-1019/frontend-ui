# Design Lab

Interactive workflow for exploring design directions through variant generation and iterative refinement.

## When to Use

- User wants to explore multiple design options
- Starting a new page/section design from scratch
- User is unsure about visual direction
- Need to generate and compare alternatives

## Process

### Step 1: Discovery Interview
Ask the user:
1. What is this interface for? (audience, purpose, context)
2. Any existing brand guidelines? (colors, fonts, tone)
3. What feeling should it evoke? (professional, playful, minimal, bold)
4. Any references or inspiration? (links, screenshots, descriptions)
5. What must it NOT look like?

### Step 2: Generate Variants
Create 3 design variants with different directions:
- **Variant A: Safe** — Familiar patterns, low risk
- **Variant B: Bold** — Distinctive, takes one real design risk
- **Variant C: Wild** — Experimental, pushes boundaries

Each variant: brief ASCII wireframe + key design decisions (colors, type, layout concept)

### Step 3: User Review
Present variants and ask:
- Which direction feels right?
- What from B/C could inform A?
- Any deal-breakers?

### Step 4: Refine
Merge feedback into a single refined direction, then build.

## Design Variant Format

```markdown
## Variant A: Safe & Professional
**Palette**: Navy (#1a1a2e) + White + Teal accent (#0ea5e9)
**Type**: Inter (body) + Playfair Display (headings)
**Layout**: Centered hero → 3 feature columns → CTA
**Signature**: Subtle gradient border on hero card
**Risk**: Low — familiar SaaS pattern, well-executed
```
